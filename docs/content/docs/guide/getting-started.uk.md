## Побачте його в дії за 5 хвилин

Вам не потрібен сервер, хостинг-провайдер чи IT-відділ. Якщо у вас є ноутбук з встановленим Docker, ви можете побачити Observer в роботі прямо зараз.

```bash
git clone https://github.com/lbrty/observer.git
cd observer
cp .env.example .env
just generate-keys
just docker-up
just run
```

Відкрийте `http://localhost:9000/health` у браузері. Якщо ви бачите `"status":"healthy"`, бекенд працює.

Потім запустіть вебінтерфейс:

```bash
just web-dev
```

Відкрийте `http://localhost:5173` — перед вами Observer.

## Що ви щойно запустили

- **Бекенд**, що обслуговує API — обробляє автентифікацію, зберігання даних та звіти
- **Базу даних** із таблицями для людей, домогосподарств, записів про підтримку, історії міграції, документів та тварин
- **Вебінтерфейс** з управлінням проєктами, рольовим контролем доступу та вбудованою звітністю
- **Автоматичну безпеку входу** — токени ротуються при кожному оновленні сесії

Все це працює на одній машині. У продакшені компілюється в один файл, який можна скопіювати на будь-який сервер.

## Готові до справжнього розгортання?

Щоб перейти від «спробувати» до «моя команда користується цим щодня», вам потрібно:

| Що | Навіщо |
| --- | --- |
| Сервер (VPS або локальний) | Observer розгортається самостійно — ваші дані ніколи не залишають вашу інфраструктуру |
| PostgreSQL | Єдиний зовнішній сервіс, який потрібен Observer |
| Приблизно 30 хвилин | Запустіть `docker compose up` на сервері, на який спрямовано ваш домен |

Жодної підписки. Жодної поосібної плати. Жодної прив'язки до постачальника. Ви володієте даними та розгортанням.

Дивіться [Розгортання](/docs/guide/deployment/) для покрокового налаштування продакшену.

## Для розробників: локальне налаштування

Якщо ви хочете працювати над самим Observer, вам знадобляться такі інструменти:

| Інструмент | Версія | Встановлення |
| --- | --- | --- |
| Go | 1.25.* | https://go.dev/dl/ |
| Bun | latest | https://bun.sh/ |
| Docker + Compose | latest | https://docs.docker.com/get-docker/ |
| Just | latest | https://github.com/casey/just#installation |

### 1. Клонування та встановлення залежностей

```bash
git clone https://github.com/lbrty/observer.git
cd observer
go mod download
bun install
```

### 2. Налаштування середовища

```bash
cp .env.example .env
```

Значення за замовчуванням працюють одразу з наданим `docker-compose.yml`. Редагування не потрібне.

### 3. Генерація ключів підпису

```bash
just generate-keys
```

Це створює пару ключів, які Observer використовує для підпису токенів входу. Каталог `keys/` додано до gitignore — кожен розробник генерує власні.

### 4. Запуск усього

```bash
just docker-up    # запускає PostgreSQL та Redis
just run          # запускає бекенд на :9000 (міграції виконуються автоматично)
just web-dev      # запускає фронтенд на :5173
```

## Щось не працює?

**Порт 5432 вже зайнятий** — Ймовірно, у вас запущений локальний PostgreSQL. Зупиніть його або змініть порт у `docker-compose.yml`.

**"no such file or directory" для шляхів до ключів** — Спочатку потрібно виконати `just generate-keys`.

**Міграція завершується помилкою "connection refused"** — Контейнер бази даних може бути ще не готовий. Зачекайте кілька секунд після `just docker-up` і спробуйте знову.
